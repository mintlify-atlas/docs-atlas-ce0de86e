## Overview

Bookmark Deeploy provides a robust bookmark management system that lets you save, organize, and manage your web links efficiently. Each bookmark contains essential metadata including URL, name, description, category, and creation timestamp.

## Data Model

Bookmarks are stored with the following structure:

```prisma
model Link {
  id          Int       @id @default(autoincrement())
  url         String
  Name        String
  Description String?   // Optional field
  category    Category  @relation(fields: [CategoryId], references: [id])
  CategoryId  Int
  createdAt   DateTime  @default(now())
}
```

<Note>
The `Description` field is optional, allowing you to create quick bookmarks without detailed notes.
</Note>

## Creating Bookmarks

### API Endpoint

Bookmarks are created via the `/link/save` endpoint. The API automatically handles category creation:

```typescript
POST /link/save

Body:
{
  "url": "https://example.com",
  "Name": "Example Site",
  "Description": "A helpful example website",
  "categoryName": "Development"
}
```

### Smart Category Handling

<Info>
When you create a bookmark, the system automatically checks if the category exists. If not, it creates the category for you.
</Info>

The API implementation follows this logic:

1. **Empty category name** → Defaults to "no category"
2. **Existing category** → Links bookmark to existing category
3. **New category** → Creates category, then links bookmark

## Viewing Bookmarks

### Fetching All Bookmarks

You can retrieve all bookmarks using the `/link/showall` endpoint:

```typescript
GET /link/showall

Response:
{
  "result": [
    {
      "id": 1,
      "url": "https://example.com",
      "Name": "Example Site",
      "Description": "A helpful example website",
      "CategoryId": 1,
      "createdAt": "2026-03-04T10:30:00Z",
      "category": {
        "CategoryName": "Development"
      }
    }
  ]
}
```

<Tip>
The API automatically includes the associated category name in the response, eliminating the need for separate category lookups.
</Tip>

## Deleting Bookmarks

### API Endpoint

Delete bookmarks using their unique ID:

```typescript
DELETE /link/delete/:id

Response:
{
  "message": "link deleted successfully",
  "deleted": { /* deleted bookmark object */ }
}
```

### Delete Implementation

The delete function automatically refreshes the bookmark list:

```tsx
const handleDelete = async () => {
  try {
    const response = await fetch(
      `https://square-forest-972c.yumnambilson.workers.dev/link/delete/${id}`,
      { method: 'DELETE' }
    );
    
    if (response.status == 204) {
      // Trigger refetch in parent component
      setrefresh((prev: number) => prev + 1);
    }
  } catch (error) {
    console.error('Error deleting bookmark:', error);
  }
};
```

<Warning>
Deleting a bookmark is permanent and cannot be undone. The system does not prompt for confirmation before deletion.
</Warning>

## State Management

The bookmark system uses React state with a refresh counter pattern:

```tsx
const [refresh, setrefresh] = useState(0)

// Trigger refetch by incrementing refresh counter
setrefresh(prev => prev + 1)
```

<Tabs>
  <Tab title="useBookmarkFilter Hook">
    ```tsx
    export function useBookmarkFilter(refresh: number = 0) {
      const [bookMark, setbookMark] = useState<response[]>([]);
      const [filterBookmark, setfilterBookmark] = useState<response[]>([]);
      
      useEffect(() => {
        fetch("https://square-forest-972c.yumnambilson.workers.dev/link/showall")
          .then(result => result.json())
          .then(jsonresponse => {
            setbookMark(jsonresponse.result);
            setfilterBookmark(jsonresponse.result);
          });
      }, [refresh]);
      
      return { bookMark, filterBookmark };
    }
    ```
