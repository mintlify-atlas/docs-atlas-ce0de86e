The Bookmark Deeploy frontend is a modern React application built with Vite, TypeScript, and Tailwind CSS. This guide covers installation, configuration, and development workflows.

## Technology Stack

- **Framework**: React 19.2.0
- **Build Tool**: Vite 7.2.4
- **Language**: TypeScript 5.9.3
- **Styling**: Tailwind CSS 4.1.18
- **Linting**: ESLint 9.39.1

<Info>
Vite provides instant server start, lightning-fast HMR (Hot Module Replacement), and optimized production builds.
</Info>

## Installation

<Steps>
  <Step title="Navigate to frontend directory">

</Steps>

## Configuration Files

### Vite Configuration

The Vite configuration is minimal and straightforward:

```typescript frontend/vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
})
```

**Plugins:**
- `@vitejs/plugin-react`: Enables React Fast Refresh and JSX transformation
- `@tailwindcss/vite`: Integrates Tailwind CSS v4 directly into Vite

## API Configuration

The frontend communicates with the Cloudflare Workers backend through REST API calls.

### Current Implementation

The API endpoint is currently hardcoded in three locations:

<CodeGroup>
```typescript hooks/useBookmarkFilter.tsx
// Fetch all bookmarks
fetch("https://square-forest-972c.yumnambilson.workers.dev/link/showall")
```

```typescript components/Card.tsx
// Delete a bookmark
fetch(`https://square-forest-972c.yumnambilson.workers.dev/link/delete/${id}`, {
  method: "DELETE"
})
```

```typescript components/Bookmark.tsx
// Save a new bookmark
fetch("https://square-forest-972c.yumnambilson.workers.dev/link/save", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(data)
})
```
</CodeGroup>

### Recommended: Use Environment Variables

<Warning>
Hardcoded URLs make it difficult to switch between development and production environments. Use Vite environment variables instead.
</Warning>

<Steps>
  <Step title="Create .env file">
    Create a `.env` file in the `frontend` directory:

    ```bash frontend/.env
    VITE_API_URL=https://square-forest-972c.yumnambilson.workers.dev
    ```
  </Step>

  <Step title="Create API utility">
    Create a centralized API configuration file:

    ```typescript src/config/api.ts
    const API_URL = import.meta.env.VITE_API_URL || 'http://localhost:8787'

    export const API_ENDPOINTS = {
      getAllLinks: `${API_URL}/link/showall`,
      saveLink: `${API_URL}/link/save`,
      deleteLink: (id: number) => `${API_URL}/link/delete/${id}`,
    }
    ```
  </Step>

  <Step title="Update components">
    Replace hardcoded URLs with the API configuration:

    ```typescript
    import { API_ENDPOINTS } from '../config/api'

    fetch(API_ENDPOINTS.getAllLinks)
    ```
  </Step>
</Steps>

## Troubleshooting

### Port already in use

If port 5173 is occupied, Vite will try the next available port. You can specify a custom port:

```bash
vite --port 3000
```

Or add it to `vite.config.ts`:

```typescript
export default defineConfig({
  plugins: [react(), tailwindcss()],
  server: {
    port: 3000
  }
})
```

### Module not found errors

Clear node_modules and reinstall:

```bash
rm -rf node_modules package-lock.json
npm install
```

### TypeScript errors

Restart your TypeScript server in your editor, or run:

```bash
tsc -b --clean
tsc -b
```

### Build fails

Check for TypeScript errors first:

```bash
npm run lint
tsc -b
```
