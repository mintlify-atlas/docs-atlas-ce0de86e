This guide walks you through setting up Bookmark Deeploy locally. You'll clone the repository, set up the database, configure environment variables, and run both the backend and frontend development servers.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v18 or higher)
- **npm** (comes with Node.js)
- **Git** for version control
- A **PostgreSQL database** (local or hosted)
- **Prisma Accelerate** connection URL (for database connection pooling)

<Warning>
  You'll need a Prisma Accelerate URL for the database connection. Sign up at [Prisma Data Platform](https://console.prisma.io/) to get your connection URL.
</Warning>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Bookmark Deeploy repository from GitHub and navigate to the project directory.
    
    ```bash
    git clone https://github.com/BilsoNyuMNAM/bookmarkDeeploy.git
    cd bookmarkDeeploy
    ```
    
    The repository contains two main directories:
    - `Backend/` - Cloudflare Worker with Hono and Prisma
    - `frontend/` - React application with Vite
  </Step>

  <Step title="Install backend dependencies">
    Navigate to the Backend directory and install the required packages.
    
    ```bash
    cd Backend
    npm install
    ```
    
    This installs:
    - `hono` - Web framework for Cloudflare Workers
    - `@prisma/client` - Database ORM client
    - `@prisma/extension-accelerate` - Connection pooling
    - `wrangler` - Cloudflare Workers CLI
    - `prisma` - Database toolkit
  </Step>

  <Step title="Configure database connection">
    Create a `.env` file in the `Backend/` directory with your database URLs.
    
    ```bash Backend/.env
    # Direct database URL for migrations
    DIRECT_URL="postgresql://user:password@host:5432/database"
    
    # Prisma Accelerate URL for the application
    DATABASE_URL="prisma://accelerate.prisma-data.net/?api_key=your_api_key"
    ```
    
    <Note>
      The `DIRECT_URL` is used for running migrations, while `DATABASE_URL` (Prisma Accelerate) is used by the application for optimal performance.
    </Note>
  </Step>

  <Step title="Set up the database schema">
    Run Prisma migrations to create the database tables.
    
    ```bash
    npx prisma migrate deploy
    ```
    
    This creates two tables:
    - **Category** - For organizing bookmarks
    - **Link** - For storing bookmark data
    
    To generate the Prisma Client:
    
    ```bash
    npx prisma generate
    ```
  </Step>

  <Step title="Configure Cloudflare Worker">
    The backend uses `wrangler.jsonc` for configuration. The default configuration should work for local development:
    
    ```jsonc Backend/wrangler.jsonc
    {
      "name": "square-forest-972c",
      "main": "src/index.ts",
      "compatibility_date": "2025-09-27",
      "compatibility_flags": ["nodejs_compat"],
      "observability": {
        "enabled": true
      }
    }
    ```
    
    For local development, you'll need to set the `DATABASE_URL` secret:
    
    ```bash
    # For local development, create a .dev.vars file
    echo 'DATABASE_URL="prisma://accelerate.prisma-data.net/?api_key=your_api_key"' > .dev.vars
    ```
    
    <Warning>
      Never commit `.dev.vars` or `.env` files to version control. These files contain sensitive credentials.
    </Warning>
  </Step>

  <Step title="Start the backend server">
    Start the Cloudflare Workers development server.
    
    ```bash
    npm run dev
    ```
    
    The backend API will be available at `http://localhost:8787`. You should see:
    
    ```json
    {
      "message": "this is the index file"
    }
    ```
    
    <Note>
      Available scripts in `Backend/package.json`:
      - `npm run dev` - Start development server
      - `npm run deploy` - Deploy to Cloudflare Workers
      - `npm test` - Run tests with Vitest
      - `npm run cf-typegen` - Generate TypeScript types
    </Note>
  </Step>

  <Step title="Install frontend dependencies">
    Open a new terminal, navigate to the frontend directory, and install dependencies.
    
    ```bash
    cd frontend
    npm install
    ```
    
    This installs:
    - `react` & `react-dom` (v19) - UI library
    - `vite` - Build tool and dev server
    - `tailwindcss` - Utility-first CSS framework
    - `typescript` - Type checking
  </Step>

  <Step title="Configure frontend API endpoint">
    Update the API endpoint in your frontend components to point to your local backend.
    
    The backend runs on `http://localhost:8787`, so API calls should use:
    
    ```typescript frontend/src/hooks/useApi.ts
    const API_BASE_URL = 'http://localhost:8787';
    
    // Example: Save a bookmark
    const response = await fetch(`${API_BASE_URL}/link/save`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(bookmarkData)
    });
    ```
  </Step>

  <Step title="Start the frontend server">
    Start the Vite development server.
    
    ```bash
    npm run dev
    ```
    
    The frontend will be available at `http://localhost:5173`.
    
    <Note>
      Available scripts in `frontend/package.json`:
      - `npm run dev` - Start development server
      - `npm run build` - Build for production
      - `npm run preview` - Preview production build
      - `npm run lint` - Run ESLint
    </Note>
  </Step>
</Steps>

## Verify the installation

<AccordionGroup>
  <Accordion title="Test the backend API">
    Open your browser or use curl to test the API endpoints:
    
    ```bash
    # Test the root endpoint
    curl http://localhost:8787
    
    # Save a test bookmark
    curl -X POST http://localhost:8787/link/save \
      -H "Content-Type: application/json" \
      -d '{
        "url": "https://example.com",
        "Name": "Test Bookmark",
        "Description": "Testing the API",
        "categoryName": "Testing"
      }'
    
    # Get all bookmarks
    curl http://localhost:8787/link/showall
    ```
  </Accordion>
  
  <Accordion title="Test the frontend">
    Navigate to `http://localhost:5173` in your browser. You should see:
    
    - A header with a button to add new bookmarks
    - A main section displaying your bookmarks organized by category
    - Filters for category and date
    - A footer
    
    Try adding a new bookmark using the UI to verify the frontend-backend connection.
  </Accordion>
</AccordionGroup>

## Common issues

<AccordionGroup>
  <Accordion title="Database connection errors">
    If you see database connection errors:
    
    1. Verify your `DATABASE_URL` is correct in `.dev.vars`
    2. Ensure your Prisma Accelerate API key is valid
    3. Check that your PostgreSQL database is accessible
    4. Run `npx prisma migrate deploy` to ensure tables are created
  </Accordion>
  
  <Accordion title="CORS errors in frontend">
    The backend includes CORS middleware, but if you encounter CORS errors:
    
    1. Verify the backend is running on `http://localhost:8787`
    2. Check that your frontend is making requests to the correct URL
    3. Ensure the CORS middleware is properly configured in `Backend/src/index.ts`
  </Accordion>
  
  <Accordion title="Port conflicts">
    If ports 8787 or 5173 are already in use:
    
    - For the backend, modify the port in your Wrangler configuration
    - For the frontend, Vite will automatically try the next available port
    - Or specify a custom port: `npm run dev -- --port 3000`
  </Accordion>
</AccordionGroup>
